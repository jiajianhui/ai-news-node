# AI 新闻抓取 Demo

这是根据 `learning-notes/ai/AI应用开发实践.md` 里的第一版思路落的一个最小 Node.js 示例：

- 用 `Express` 提供本地网页和接口
- 抓取多个 RSS 来源
- 用关键词筛选 AI 相关新闻
- 去重后生成一份简日报
- 保存到本地 `data/latest-digest.json`

当前版本还是一个偏底层、偏工程验证的 demo。它先解决的是“新闻能不能稳定抓到、解析出来、筛出来、落下来”这件事，而不是一步到位做成真正的 AI 新闻助理。

## 运行环境

- `Node.js 22.12.0 - 24.x`
- `npm 10+`

## 启动方式

```bash
cd /Users/jiajianhui/Desktop/cx/ai-news-node
npm install
npm start
```

启动后打开：

```text
http://localhost:3000
```

## 接口

- `GET /api/health`：健康检查
- `GET /api/news/latest`：读取最近一次日报
- `POST /api/news/run`：立即抓取一次

## 当前版本解决了什么

这版项目主要验证 4 件事：

- 能否稳定抓取多个 RSS / Atom 新闻源
- 能否把 XML 解析成统一的 JS 对象结构
- 能否通过关键词做一层基础筛选
- 能否生成可复用的 digest 结果供前端或后续系统读取

这一步看起来不够“智能”，但它是后续所有自动摘要、个性化订阅、定时推送能力的底座。

## 未来发展方向

这个项目更合理的演进方式，不是直接把所有 AI 能力一口气塞进去，而是分阶段往上长：

### 第一阶段：可用的新闻处理引擎

也就是当前这版重点在做的事情：

- 固定新闻源
- 固定关键词
- 手动触发或通过接口触发
- 生成结构化 digest

目标是把抓取、解析、过滤、去重、存储这条链路做稳。

### 第二阶段：半自动新闻助手

在当前版本之上，可以继续增加：

- 定时任务，比如每天早上自动生成日报
- 更丰富的摘要能力，比如调用 LLM 生成更像人写的总结
- 多渠道推送，比如 Telegram、飞书、企业微信、邮件
- 更好的持久化，比如 SQLite 或 PostgreSQL
- 更灵活的配置，比如不同关键词组合、不同新闻源分组

这一阶段的目标是从“能跑”变成“能每天用”。

### 第三阶段：面向个人的 AI 新闻代理

终极形态不应该只是“预设关键词抓新闻”，而应该更接近下面这条链路：

```text
用户直接输入关注方向 / 提示词
-> 系统理解用户意图
-> 自动选择或扩展信息源
-> 定时抓取、筛选、排序
-> 生成个性化摘要
-> 主动推送给用户
```

比如用户可以直接说：

```text
每天早上给我 5 条 AI agent、OpenAI、Anthropic 相关的重要动态，不要泛科技新闻，也不要融资八卦。
```

系统再把这句话拆成真正可执行的订阅规则、筛选逻辑和推送任务。

## 终极形态

如果把这个项目继续往前做，它最终更像一个“个人 AI 新闻助理”，而不只是一个 RSS 抓取脚本。

理想状态下，它应该具备这些能力：

- 用户用自然语言描述关注主题，而不是手工改代码里的关键词
- 系统长期记住用户偏好，而不是每次重新配置
- 系统自动定时运行，而不是手动点接口
- 输出的不是标题拼接，而是真正有筛选和判断的摘要
- 不只是“抓到了什么”，还包括“为什么值得看”
- 最终结果可以按日报、周报、专题追踪等不同形式交付

## 你可以继续扩展的方向

- 接 OpenAI API 做自动摘要
- 加 `node-cron` 做每天 8 点自动跑
- 接企业微信、飞书、Telegram 做推送
- 改成 SQLite 或 PostgreSQL 持久化
